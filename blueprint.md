# Project Name: "SwiftTool" - All-in-One Browser Utility
## Vision
A fast, free, no-login-required utility site where all processing happens 100% in the user's browser.

## Core Rules
1. **Zero Server Costs:** Use client-side libraries (ffmpeg.wasm, pdf-lib, canvas) so I don't pay for processing power.
2. **Ad-Ready Layout:** Leave specific "Safe Zones" in the UI for Google AdSense banners (Top, Sidebar, and Download page).
3. **No Database:** Use browser LocalStorage to show users their "Recent Tools" so they come back daily.

## Initial Toolset (V1)
1. **PDF Splitter:** Split one PDF into multiple pages.
2. **Image Converter:** Change JPG/PNG to WebP (for SEO) or vice-versa.
3. **Background Remover:** Using a browser-based AI model (like @imgly/background-removal).

# SwiftTool UI/UX Blueprint
## Design Language: "Gen-Z Utility"
- **Vibe:** Bold, Clean, Playful (inspired by remove.bg & Red Flag Gen Z project).
- **Colors:** Deep Indigo (#4F46E5), Pure White, and Soft Gray (#F9FAFB).
- **Components:** Large "Bento-Box" style grid for tools, massive rounded buttons (2xl), and "Glassmorphism" cards.

## Monetization Layout
- **Hero Ad:** 728x90px container directly below the main "Headline."
- **Sidebar Ad:** 300x250px square next to the "Processing" area.
- **Post-Action Ad:** A dedicated ad spot on the "Success/Download" screen.
